📚 수능 학습 콘텐츠 HTML 변환 작업 매뉴얼

버전 1.0 | 기준: 2027 수능완성 사용설명서(유형편) + 문제 완성본 PDF | 작성일: 2026.06.12

🎯 매뉴얼의 목적
본 매뉴얼은 AI(Claude 등)와 인간 협업자가 『수능완성 사용설명서 유형편』 및 『수능 완성 문제 완성본』 PDF를 원본 그대로 HTML로 변환할 수 있도록,
모든 규칙, 형식, 기술적 요구사항, 검증 절차를 체계화한 문서입니다.
한 글자도 다르지 않은 원문 복원 + 2단 분석 + 상세 오답 해설이 핵심입니다.

📌 0. 작업의 핵심 조건

  • 원본 PDF는 두 종류: 사용설명서(지문+분석+해제) + 문제 완성본(지문+문제+선지+정답+해설)
  • 출력 결과물은 반드시 원문과 100% 일치 (OCR 오류 교정 시에도 의미 훼손 금지)
  • 2단 좌우 레이아웃: 왼쪽(원문), 오른쪽(분석/해설)
  • 시구/대사 단위 분리 + 밑줄(ⓐ~ⓔ, ㉠~㉤, [A], [B]) 표시
  • 각 선택지별 상세 오답 이유 (✅ 정답 / ❌ 오답)
  • 반응형 웹 (모바일에서 2단 → 세로)
  • 파일명 한글: [작품명/주제]_[유형].html

📂 1. 원문 텍스트 확보 및 정제

  • PDF → 텍스트 추출 후 OCR 오류 교정 (예: ㉠이 ①로 인식됨 → 복원)
  • 자주 발생하는 오류 및 교정 예시:
OCR 오인식올바른 원문비고
①㉠밑줄 기호
천지천치시어
꽃살구풋살구시어
하드라하드라방언 그대로
호수입식 책전립이호수입식 책전립이한자어 복원

🧱 2. HTML 기본 구조 (템플릿)

<!DOCTYPE html>
<html lang="ko">
<head>
<meta charset="UTF-8">
<meta name="viewport" content="width=device-width, initial-scale=1.0">
<title>유형 제목 - 작품명/주제 (통합본)</title>
<style>/* CSS 생략(본문 참조) */</style>
</head>
<body>
<div class="container">
<div class="header">...</div>
<div class="two-columns">...</div>
<div class="questions-section">...</div>
</div>
</body>
</html>

📝 3. 왼쪽 원문 영역(지문) 상세 구성

3.1 독서 지문 (사회·과학·기술·인문)

<p>... <span class="underline-phrase">ⓐ요건</span>으로 하고 있으며 ...</p>

3.2 문학 작품

  • 시: 각 시구를 .poem-line으로 나누고, 해설은 .line-analysis (← 기호 사용)
  • 소설/희곡/고전: .story-line 단위 분리, 중요 발문은 .line-analysis
<div class="poem-line">
<span class="line-text"><span class="underline-phrase">㉠애비는 종이었다.</span> 밤이 깊어도 오지 않았다.</span>
<div class="line-analysis">← 화자의 태생적 한계 – 종의 자식으로 태어난 운명</div>
</div>

📌 4. 오른쪽 분석 영역 구성

  • .analysis-box – 문단별 분석, 인물 분석, 핵심 개념
  • .vocab-box – 어구 풀이 (연두 테두리)
  • .info-box – 독해 포인트, 주제, ‘이것만은 폭 익히지!’
  • .highlight-table – 비교 표

분석 내용은 사용설명서의 핵심 문장을 요약·정리하되, 원문 의미를 훼손하지 않아야 함.

🧪 5. 문제 + 해설 영역 (매우 중요)

<div class="question-full">
<div class="q-num">1번</div>
<div class="question-stem">(문제 지문 원문)</div>
<div class="choice-item">
<div class="choice-text">① ...</div>
<div class="choice-analysis">❌ <strong>오답</strong> – 이유</div>
</div>
<div class="correct-badge">📍 ①번이 정답</div>
</div>
  • 정답 선택지: ✅ <strong>정답</strong> – (근거)
  • 오답 선택지: ❌ <strong>오답</strong> – (이유) 또는 ❌ <strong>오답 아님 (적절함)
  • 보기: #eef2ff 배경 박스에 원문 그대로 삽입

🎨 6. 스타일 규칙 핵심

  • 2단: .two-columns { display: flex; }, @media (max-width: 900px) { flex-direction: column; }
  • 밑줄: .underline-phrase { background: #fcf4d9; border-bottom: 2px solid #e67e22; }
  • 시구 해설: .line-analysis { border-left: 2px solid #e67e22; font-family: monospace; color: #2c5282; }
  • 문제 영역: .question-full { border: 1px solid #cfdfed; border-radius: 26px; }

🔍 7. 최종 검토 및 원문 비교 (필수 단계)

✅ 비교할 항목 (반드시 확인)
  • 지문 전체 (문단 수, 문장 순서, 띄어쓰기, 특수문자, 각주)
  • 밑줄 위치 (ⓐ~ⓔ, ㉠~㉤, [A], [B])
  • 문제 지문 (번호, 내용)
  • 선택지 (①~⑤, 내용 순서)
  • 보기 (전체 텍스트)
  • 정답 표시
  • 오답 해설 (지문 근거와 일치)
⚠️ 중요
HTML 완성 후 반드시 PDF 원문을 옆에 열고 한 줄씩 비교하세요.
한 글자라도 다르면 수정해야 합니다.

📄 8. 파일 저장 및 전달

  • 파일명: [작품명/주제]_[유형].html (예: 정도전_유교_불교_비판.html)
  • 인코딩: UTF-8
  • 공유 시 HTML 파일 그대로 전달 (별도 리소스 없음)

🧭 9. 프롬프트 템플릿 (다른 AI에게 지시할 때)

다음은 '수능완성 사용설명서(유형편)'과 '수능 완성 문제 완성본' PDF의 원문입니다. 아래 규칙에 따라 완전한 HTML 파일로 변환해 주세요. 【규칙】 1. 2단 좌우 레이아웃 (왼쪽: 원문, 오른쪽: 분석) 2. 밑줄(ⓐ~ⓔ, ㉠~㉤)은 노란 밑줄(.underline-phrase)로 표시 3. 문학 작품은 시구/대사 단위로 분리하고, 각 구절 아래에 상세 해설(← 기호 사용) 추가 4. 문제 영역은 선택지별로 오답 이유를 상세히 작성 (✅ 정답, ❌ 오답) 5. 보기가 있으면 원문 그대로 배경 박스에 삽입 6. 반응형 웹 (모바일에서 2단 세로 전환) 7. 파일명은 [작품명_주제.html] 한글 저장 8. CSS는 내부 스타일로 작성 9. 완성 후 반드시 원문과 비교하여 누락·오탈자 확인 (한 글자도 달라서는 안 됨) 【원문】 [여기에 지문 + 문제 + 보기 전체를 붙여넣기]

✅ 10. 최종 체크리스트 (출력 전 반드시 확인)

  • ☐ 지문에 모든 밑줄(ⓐ~ⓔ, ㉠~㉤)이 빠짐없이 표시되었는가?
  • ☐ 원문의 문단 구분이 유지되었는가?
  • ☐ 각 문제의 선택지가 PDF 순서와 동일한가?
  • ☐ 각 선택지에 ✅ 또는 ❌ 표시와 함께 상세한 오답 이유가 있는가?
  • ☐ 보기가 있다면 원문 그대로이며, 배경색이 적용되었는가?
  • ☐ 오른쪽 분석 박스가 비어 있지 않고, 사용설명서의 핵심 내용을 담고 있는가?
  • ☐ 반응형 테스트 (브라우저 너비 줄였을 때 깨지지 않는가?)
  • ☐ 파일명이 한글로 명확하고, 저장 시 UTF-8 인코딩인가?

📚 수능 학습 콘텐츠 HTML 변환 작업 매뉴얼

버전 2.0 | 기준: 2027 수능완성 사용설명서(유형편) + 문제 완성본 PDF | 작성일: 2026.07.03

🎯 매뉴얼의 목적
본 매뉴얼은 AI(Claude 등)와 인간 협업자가 『수능완성 사용설명서 유형편』 및 『수능 완성 문제 완성본』 PDF를 원본 그대로 HTML로 변환할 수 있도록,
모든 규칙, 형식, 기술적 요구사항, 검증 절차를 체계화한 문서입니다.
한 글자도 다르지 않은 원문 복원 + 2단 분석 + 상세 오답 해설이 핵심입니다.
⚠️ 가장 중요한 원칙
“원문 그대로”는 절대적인 기준입니다. 지문, 문제, 선택지, 보기 모두 PDF와 한 글자도 달라서는 안 됩니다.
AI가 “내용이 같다”, “의미가 같다”고 판단하더라도 문장이 다르면 불합격입니다.

📌 0. 작업의 핵심 조건

  • 원본 PDF는 두 종류: 사용설명서(지문+분석+해제) + 문제 완성본(지문+문제+선지+정답+해설)
  • 출력 결과물은 반드시 원문과 100% 일치 (OCR 오류 교정 시에도 의미 훼손 금지)
  • 2단 좌우 레이아웃: 왼쪽(원문), 오른쪽(분석/해설)
  • 시구/대사 단위 분리 + 밑줄(ⓐ~ⓔ, ㉠~㉤, [A], [B]) 표시
  • 각 선택지별 상세 오답 이유 (✅ 정답 / ❌ 오답)
  • 반응형 웹 (모바일에서 2단 → 세로)
  • 파일명 한글: [작품명/주제]_[유형].html

📂 1. 원문 텍스트 확보 및 정제

  • PDF → 텍스트 추출 후 OCR 오류 교정 (예: ㉠이 ①로 인식됨 → 복원)
  • 자주 발생하는 오류 및 교정 예시:
OCR 오인식올바른 원문비고
①㉠밑줄 기호
천지천치시어
꽃살구풋살구시어
하드라하드라방언 그대로
호수입식 책전립이호수입식 책전립이한자어 복원

📝 3. 왼쪽 원문 영역(지문) 상세 구성

3.1 독서 지문 (사회·과학·기술·인문)

<p>... <span class="underline-phrase">㉠초란색</span> 개념은 예측이 엇갈리는 문제를 드러내어 ...</p>
📌 독서 지문 처리 시 주의사항
  • 문단 구분을 반드시 유지할 것 (PDF의 문단 나누기 그대로)
  • 밑줄(㉠~㉤, ⓐ~ⓔ)은 .underline-phrase 클래스로 표시
  • 지문 마지막 문단은 절대 축약하지 말 것 – 원문 그대로 복사

3.2 문학 작품

  • 시: 각 시구를 .poem-line으로 나누고, 해설은 .line-analysis (← 기호 사용)
  • 소설/희곡/고전: .story-line 단위 분리, 중요 발문은 .line-analysis
<div class="poem-line">
<span class="line-text"><span class="underline-phrase">㉠애비는 종이었다.</span> 밤이 깊어도 오지 않았다.</span>
<div class="line-analysis">← 화자의 태생적 한계 – 종의 자식으로 태어난 운명</div>
</div>

📌 4. 오른쪽 분석 영역 구성

  • .analysis-box – 문단별 분석: 각 문단의 핵심 내용을 사용설명서 기반으로 정리
  • .vocab-box – 어구 풀이: 핵심 용어 정의 (연두 테두리)
  • .info-box – 독해 포인트: 글의 구성, 주제, 핵심 논리 포인트
  • .highlight-table – 비교 표: 개념 비교, 철학자별 입장 비교 등
  • .info-box[style*="fef2e0"] – 수능 필수 체크: ‘이것만은 꼭 익히자!’
⚠️ 분석 작성 시 주의
분석 내용은 사용설명서의 핵심 문장을 요약·정리하되, 원문 의미를 훼손하지 않아야 합니다.
사용설명서에 없는 내용을 임의로 추가하지 마세요.

🧭 4-1. 구조도 (필수 항목)

📌 구조도는 반드시 포함해야 합니다.
지문의 글 전개 흐름을 한눈에 파악할 수 있도록 구조도(Structure Diagram)를 추가하세요.
  • 구조도는 .structure-wrapper 안에 .structure-grid로 구성
  • 각 카드(.struct-card)는 글의 단계별 흐름을 나타냄
  • 카드 사이에 .arrow-symbol로 진행 방향 표시 (➡️)
  • 각 카드에는 해당 문단의 핵심 내용과 문단 번호를 함께 표시
  • 하단에 전체 흐름 요약을 한 줄로 추가
📌 구조도 예시 (귀납의 정당화 지문)
① 귀납 정의 & 흄·러셀 → ② 굿맨의 비판 & 초란색 → ③ 스윈번의 기준 & 굿맨 재비판
📍 전체 흐름 : 귀납 개념 → 흄(회의) → 러셀(조건부 옹호) → 굿맨 비판(초란색) → 스윈번 대안(두 기준) → 굿맨 재비판(기준 자의성)

🧪 5. 문제 + 해설 영역 (매우 중요)

⚠️ 문제 및 선택지는 PDF 원문과 반드시 1:1로 일치해야 합니다.
절대 선택지를 재작성하거나 순서를 바꾸지 마세요.
PDF의 ①~⑤번 선택지를 그대로 복사하여 붙여넣기하세요.
<div class="question-full">
<div class="q-num">01</div>
<div class="question-stem">(문제 지문 원문)</div>
<div class="choice-item correct">
<div class="choice-text">① ... (PDF 원문 그대로)</div>
<div class="choice-analysis">✅ <strong>정답</strong> – PDF 해설 내용</div>
</div>
<div class="choice-item wrong">
<div class="choice-text">② ... (PDF 원문 그대로)</div>
<div class="choice-analysis">❌ <strong>오답</strong> – PDF 해설 내용</div>
</div>
<div class="correct-badge">📍 정답 ①번</div>
</div>
  • 정답 선택지: .choice-item.correct + ✅ 정답 – (근거)
  • 오답 선택지: .choice-item.wrong + ❌ 오답 – (이유)
  • 보기: .view-box 클래스에 PDF 원문 그대로 삽입
  • 문항 번호: PDF의 실제 번호와 일치해야 함 (001번, 002번 등)

📋 5-1. 문제 유형별 처리

유형처리 방법예시
일반 선택형 ①~⑤ 선택지를 PDF 원문 그대로 복사 “윗글에 대한 이해로 적절하지 않은 것은?”
ㄱ~ㄹ / 보기 보기 박스(.view-box)에 원문 삽입, 선택지는 PDF 순서 유지 “다음의 ㄱ~ㄹ에 대해 판단한 것으로...”
빈칸 추론 표 형태로 A, B에 들어갈 말을 선택 “A와 B에 들어갈 말을 바르게 짝지은 것은?”
어휘/문맥 밑줄 친 단어의 의미를 묻는 문제, PDF 선택지 그대로 “㉠의 의미로 가장 적절한 것은?”

🔍 6. 최종 검토 및 원문 비교 (필수 단계)

✅ 비교할 항목 (반드시 확인)
  • ☐ 지문 전체 (문단 수, 문장 순서, 띄어쓰기, 특수문자, 각주)
  • ☐ 밑줄 위치 (ⓐ~ⓔ, ㉠~㉤, [A], [B])
  • ☐ 지문 마지막 문단이 축약되지 않았는지 (반드시 원문 그대로)
  • ☐ 문제 지문 (번호, 내용)
  • ☐ 선택지 (①~⑤, 내용 순서 – 재작성 금지)
  • ☐ 보기 (전체 텍스트)
  • ☐ 정답 표시
  • ☐ 오답 해설 (지문 근거와 일치)
  • ☐ 문항 번호가 PDF와 일치하는지 (001번이 누락되면 안 됨)
⚠️ 자주 발생하는 오류
  • 지문 축약: 마지막 문단을 요약하지 말고 원문 그대로 복사
  • 선택지 재작성: “내용이 같다”고 생각해도 문장이 다르면 오류
  • 문항 누락: PDF의 모든 문항(001번부터)이 포함되었는지 확인
  • 문항 순서 변경: PDF 순서 그대로 유지
  • 분석 내용 임의 추가: 사용설명서에 없는 내용을 넣지 말 것
  • 구조도 누락: 구조도는 필수 항목
✅ 검증 완료 사례: 실전 2회 01~05 (귀납의 정당화)
본 매뉴얼의 규칙에 따라 다음 항목을 모두 확인 완료:
  • 지문: 흄·러셀·굿맨·스윈번·초란색 논쟁 – 원문 100% 일치
  • 001~005번 문항 – PDF 순서 및 선택지 완전 일치
  • 001번: “윗글을 통해 답을 찾을 수 없는 질문은?” – 선택지 원문 복원
  • 003번: “㉠의 의미로 가장 적절한 것은?” – 선택지 원문 복원
  • 구조도 포함 – 글 전개 흐름 시각화
  • 사용설명서 분석 – 오른쪽 열에 문단별 분석·어구 풀이·독해 포인트 포함

📄 7. 파일 저장 및 전달

  • 파일명: [작품명/주제]_[유형].html (예: 귀납의_정당화_01~05.html)
  • 인코딩: UTF-8
  • 공유 시 HTML 파일 그대로 전달 (별도 리소스 없음)

🧭 8. 프롬프트 템플릿 (다른 AI에게 지시할 때)

다음은 '수능완성 사용설명서(유형편)'과 '수능 완성 문제 완성본' PDF의 원문입니다. 아래 규칙에 따라 완전한 HTML 파일로 변환해 주세요. 【규칙】 1. 2단 좌우 레이아웃 (왼쪽: 원문, 오른쪽: 분석) 2. 밑줄(ⓐ~ⓔ, ㉠~㉤)은 노란 밑줄(.underline-phrase)로 표시 3. 문학 작품은 시구/대사 단위로 분리하고, 각 구절 아래에 상세 해설(← 기호 사용) 추가 4. 문제 영역은 선택지별로 오답 이유를 상세히 작성 (✅ 정답, ❌ 오답) 5. 보기가 있으면 원문 그대로 배경 박스에 삽입 6. 반응형 웹 (모바일에서 2단 세로 전환) 7. 파일명은 [작품명_주제.html] 한글 저장 8. CSS는 내부 스타일로 작성 9. 구조도(Structure Diagram)를 반드시 포함 (글 전개 흐름을 카드 형태로 시각화) 10. 지문, 문제, 선택지는 PDF와 100% 일치 – 한 글자도 달라서는 안 됨 (축약·재작성 금지) 11. 문항 번호는 PDF 순서 그대로 (001번부터 누락 없이) 【원문】 [여기에 지문 + 문제 + 보기 전체를 붙여넣기]

✅ 9. 최종 체크리스트 (출력 전 반드시 확인)

  • ☐ 지문에 모든 밑줄(ⓐ~ⓔ, ㉠~㉤)이 빠짐없이 표시되었는가?
  • ☐ 지문이 PDF와 한 글자도 다르지 않은가? (마지막 문단 포함)
  • ☐ 원문의 문단 구분이 유지되었는가?
  • ☐ 각 문제의 선택지가 PDF 순서와 내용까지 동일한가? (재작성 금지)
  • ☐ 문항 번호가 PDF와 일치하는가? (누락된 번호는 없는가?)
  • ☐ 각 선택지에 ✅ 또는 ❌ 표시와 함께 상세한 오답 이유가 있는가?
  • ☐ 보기가 있다면 원문 그대로이며, 배경색이 적용되었는가?
  • ☐ 오른쪽 분석 박스가 비어 있지 않고, 사용설명서의 핵심 내용을 담고 있는가?
  • ☐ 구조도가 포함되어 있는가? (글 전개 흐름을 시각화)
  • ☐ 반응형 테스트 (브라우저 너비 줄였을 때 깨지지 않는가?)
  • ☐ 파일명이 한글로 명확하고, 저장 시 UTF-8 인코딩인가?
